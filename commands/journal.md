Update the project journal for today's session.

## Steps

1. **Read the current journal**: Read `JOURNAL.md` at the project root. If it doesn't exist, create it with a header.

2. **Review the conversation**: Look at everything discussed in this session — decisions made, work completed, problems encountered, direction changes, ideas explored.

3. **Write today's entry** at the TOP of the journal (newest first), using this format:

```markdown
## YYYY-MM-DD — [short theme/title]

### What happened
- [Bullet points of key events, decisions, and work done]

### Decisions made
- [Any directional choices, scope changes, or architectural decisions]

### What's next
- [Concrete next steps for the next session]

### Open questions
- [Anything unresolved that needs thinking or research]
```

4. **Keep it honest and concise** — this is a personal development journal, not marketing copy. Include failed approaches, pivots, and real reasoning. Skip fluff.

5. **Don't duplicate** — if today already has an entry, update it rather than creating a new one.

Rules:
- Newest entries at the top, oldest at the bottom
- Use the actual date, not relative dates
- Be specific about what was built, not just what was discussed
- Include links to relevant files when useful
